Releases Thunderous follows Semantic Versioning, which means that the version number indicates the nature of the changes in the release. The version number is formatted as MAJOR.MINOR.PATCH, where: MAJOR is incremented for incompatible API changes, MINOR is incremented for added functionality in a backwards-compatible manner, and PATCH is incremented for backwards-compatible bug fixes.
  • 1.1.1 (Latest) Added a permanent fix for the temporary workaround in rendering, and added importNode() for more stable behavior when rendering fragments.
  • 1.1.0 Adds support for rendering nested fragments, arrays of fragments, and reactive fragments via derived(). Read more about rendering.
  • 1.0.2 Fixes bugs with ElementResult.define() and RegistryResult.define() where the elements were not being defined/tracked correctly.
  • 1.0.1 Fixes a bug where the insertTemplates function would parse attribute values with spaces incorrectly.
  • 1.0.0 Latest major release of Thunderous. This release introduces server-side rendering support.
  • 0.6.4 Release notes for versions 0.6.4 and under are not available. These versions were relatively experimental and unstable.

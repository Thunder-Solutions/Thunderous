Registries The createRegistry() function is used to create a registry for custom elements. <%= include('_code-snippets/registry.ejs') %> This allows you to delegate custom element definitions to the consumer of your library, by using their associated classes to look up tag names dynamically. This can be useful when you need to select a custom element whose tag name is variable. <%= include('_code-snippets/registry-tag-name.ejs') %> This is not to be confused with scoped element registries, which allow multiple definitions for a single tag name to exist in different shadow roots. This is a global registry for tracking custom elements and their tag names. A feature to support scoped registries is currently under development.
